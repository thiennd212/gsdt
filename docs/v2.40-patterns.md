## Dapper Row-Level Security (RLS) Pattern

All Dapper queries accessing tenant-sensitive data MUST filter by tenant ID. Use helper extension for consistency:

```csharp
// In IReadDbConnection extensions
public static IEnumerable<T> QueryWithTenantFilter<T>(
    this IReadDbConnection conn,
    string sql,
    Guid tenantId,
    object? param = null)
{
    // Append tenant filter to WHERE clause
    var fullSql = $"{sql} AND TenantId = @TenantId";
    var parameters = new DynamicParameters(param);
    parameters.Add("TenantId", tenantId);
    return conn.Query<T>(fullSql, parameters);
}
```

**Session Context Enforcement (SQL Server):**
Every database connection executes `sp_set_session_context` via `TenantSessionContextInterceptor`:

```sql
EXEC sp_set_session_context 'TenantId', @tenantId
```

RLS policy at table layer:
```sql
CREATE SECURITY POLICY fn_TenantFilter 
    ADD FILTER PREDICATE fn_TenantFilter(TenantId)
    ON [AuditLog], [Cases], [Forms], ...
```

## Backend Multi-Async Pattern: Task.WhenAll with Error Collection

For operations that should complete even if individual items fail, use `Task.WhenAll` with proper error handling:

```csharp
// ✓ Correct — collects all results including errors
try
{
    var results = await Task.WhenAll(
        deleteConsents.Select(c => consentClient.DeleteAsync(c.Id))
    );
}
catch (Exception ex)
{
    // Log partial failures; continue with successful deletions
}

// ✗ Incorrect — stops on first failure
foreach (var consent in deleteConsents)
{
    await consentClient.DeleteAsync(consent.Id);
}
```

Apply to:
- Bulk deletions (20 pages: ConsentController, CaseCommentController, etc.)
- Cleanup operations (SignalR ref-counting, Hangfire cleanup)
- Multi-tenant teardown scenarios

**Frontend equivalent in React:**
```tsx
const results = await Promise.allSettled(
  selectedItems.map(id => mutation({ id }))
);

// Handle both fulfilled and rejected results
results.forEach((result, idx) => {
    if (result.status === 'rejected') {
        console.error(`Item ${idx} failed:`, result.reason);
    }
});
```

## Tenant GUID Validation

All tenant IDs from request params/body MUST be validated as non-empty GUIDs:

```csharp
public static Result ValidateTenantId(Guid tenantId)
{
    if (tenantId == Guid.Empty)
        return Result.Fail(new ValidationError("Tenant ID cannot be empty."));
    return Result.Ok();
}
```

Apply validation in command validators:

```csharp
public class CreateCaseCommandValidator : AbstractValidator<CreateCaseCommand>
{
    public CreateCaseCommandValidator()
    {
        RuleFor(x => x.TenantId)
            .NotEmpty().WithMessage("Mã đơn vị không được để trống.")
            .Must(tid => tid != Guid.Empty);
    }
}
```

## Frontend Pagination Reset on Search/Filter

When search/filter state changes, reset pagination to page 1:

```jsx
const handleSearch = (searchText) => {
    setSearchTerm(searchText);
    setPagination({ ...pagination, current: 1 });  // Reset to page 1
    refetch();  // Re-fetch with new search
};

const handleFilter = (selectedStatus) => {
    setFilterStatus(selectedStatus);
    setPagination({ ...pagination, current: 1 });  // Reset to page 1
};
```

**useServerPagination hook pattern:**
```tsx
const { 
  data, 
  pagination, 
  setPagination,
  refetch 
} = useServerPagination({
  queryFn: async (page, pageSize) => {
    return getCases({ page, pageSize, status: filterStatus });
  },
  resetDeps: [filterStatus, searchTerm],  // Reset on these changes
});
```

## Client-Side Search: Fetch-All Pattern (v2.40)

For pages with moderate data (300–5000 records), fetch entire dataset once and filter client-side instead of paginating server-side.

**Benefits:**
- Better UX for filtering/sorting (no network latency per action)
- Reduced server load
- Local search responsiveness

**Implementation:**

```tsx
// Query hook
const { data: allCases } = useCasesQuery({ 
    tenantId, 
    skipPagination: true  // Fetch all
});

// Client-side search/filter
const [searchText, setSearchText] = useState('');
const filtered = useMemo(
  () => allCases?.filter(c => 
      c.title.toLowerCase().includes(searchText.toLowerCase())
  ) || [],
  [allCases, searchText]
);

// Pagination applied client-side
const [page, setPage] = useState(1);
const pageSize = 20;
const paginatedData = useMemo(
  () => filtered.slice((page - 1) * pageSize, page * pageSize),
  [filtered, page, pageSize]
);
```

**When to use:**
- Admin dashboards with stable reference data (users, params, etc.)
- Form lists with stable definitions
- Search pages with moderate result sets (<5000)

**When NOT to use:**
- Tables with millions of rows
- Real-time data that changes frequently
- Heavy filtering/sorting logic

Applied to 26 pages in v2.40 (Forms, Organization, Cases search, etc.)

## SignalR Auth Token Validation

Ensure WebSocket connections validate auth tokens on hub connection:

```csharp
public class NotificationHub : Hub
{
    private readonly ICurrentUser _currentUser;
    
    public override async Task OnConnectedAsync()
    {
        // Validate token on connection
        var token = Context.GetHttpContext()?.Request.Headers["Authorization"];
        if (string.IsNullOrEmpty(token))
        {
            Context.Abort();
            return;
        }
        
        await base.OnConnectedAsync();
    }
    
    public async Task SubscribeToTask(Guid taskId)
    {
        var currentUserId = _currentUser.Id;
        var groupName = $"task-{taskId}";
        await Groups.AddToGroupAsync(Context.ConnectionId, groupName);
    }
}
```

**v2.40 Fixes:**
- Auth token validation on hub connection (prevents unauthenticated subscriptions)
- Ref-counting to prevent duplicate-subscribe race conditions
- Pagination reset on search/filter to prevent stale results

## Open Redirect Prevention

Validate redirect URLs to prevent open redirect attacks:

```csharp
public static bool IsLocalRedirect(string? redirectUrl)
{
    if (string.IsNullOrEmpty(redirectUrl))
        return false;
    
    // Block protocol-relative URLs (//evil.com)
    if (redirectUrl.StartsWith("//"))
        return false;
    
    // Block absolute URLs to other hosts
    if (Uri.TryCreate(redirectUrl, UriKind.Absolute, out var uri))
        return Uri.IsWellFormedUriString(uri.GetLeftPart(UriPartial.Authority), UriKind.Absolute) &&
               uri.Host == HttpContext.Request.Host.Host;
    
    // Allow relative URLs
    return Uri.IsWellFormedUriString(redirectUrl, UriKind.Relative);
}
```

Use in auth flows:

```csharp
[HttpPost("login")]
public async Task<IActionResult> Login(LoginRequest req)
{
    var result = await Mediator.Send(new LoginCommand(req.Email, req.Password));
    
    var redirectUrl = req.ReturnUrl ?? "/dashboard";
    if (!IsLocalRedirect(redirectUrl))
        redirectUrl = "/dashboard";  // Fall back to safe default
    
    return LocalRedirect(redirectUrl);
}
```

## Outbox Interceptor: Event Preservation

Prevent domain event loss during transaction failures:

```csharp
public class OutboxInterceptor : SaveChangesInterceptor
{
    public override async ValueTask<InterceptionResult> SavingChangesAsync(
        DbContextEventData eventData,
        InterceptionResult result,
        CancellationToken cancellationToken = default)
    {
        var context = eventData.Context!;
        
        // Extract domain events before SaveChanges
        var domainEvents = context.ChangeTracker
            .Entries<IAggregateRoot>()
            .SelectMany(e => e.Entity.GetDomainEvents())
            .ToList();
        
        // Publish to outbox table (transactional insert)
        var outboxEntries = domainEvents
            .Select(e => new OutboxEntry(Guid.NewGuid(), e.EventType, JsonSerializer.Serialize(e)))
            .ToList();
        
        context.Set<OutboxEntry>().AddRange(outboxEntries);
        
        return await base.SavingChangesAsync(eventData, result, cancellationToken);
    }
}
```

Background job processes outbox and publishes to event bus:

```csharp
public class PublishOutboxEventsJob
{
    private readonly IReadDbConnection _conn;
    private readonly IEventBus _eventBus;
    
    public async Task Execute()
    {
        var entries = await _conn.QueryAsync<OutboxEntry>(
            "SELECT * FROM OutboxEntries WHERE ProcessedAt IS NULL");
        
        foreach (var entry in entries)
        {
            var @event = JsonSerializer.Deserialize(entry.EventData);
            await _eventBus.PublishAsync(@event);
            
            await _conn.ExecuteAsync(
                "UPDATE OutboxEntries SET ProcessedAt = GETUTC() WHERE Id = @Id",
                new { entry.Id });
        }
    }
}
```

## BackgroundJobTenantContext

Ensure Hangfire background jobs execute with correct tenant context:

```csharp
public class BackgroundJobTenantContext
{
    private static readonly AsyncLocal<Guid> _tenantId = new();
    
    public static Guid TenantId => _tenantId.Value;
    
    public static IDisposable SetTenantId(Guid tenantId)
    {
        var previousValue = _tenantId.Value;
        _tenantId.Value = tenantId;
        return new DisposableAction(() => _tenantId.Value = previousValue);
    }
}

// In Hangfire background job
public class ProcessWorkflowTransitionsJob
{
    public async Task Execute(Guid tenantId)
    {
        using (BackgroundJobTenantContext.SetTenantId(tenantId))
        {
            // All database calls execute with correct tenant isolation
            var transitions = await _workflowService.GetPendingTransitionsAsync();
            await _workflowService.ExecuteTransitionsAsync(transitions);
        }
    }
}
```
